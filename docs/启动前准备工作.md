• 需要做两类：准备工作 和 项目初始化工作。

  ———

  ## 一、正式开发前准备

  ### 1. 确认底层 CLI 能力

  必须先跑一遍：

  npx skills --help
  npx add-mcp --help

  重点确认：

  - 是否支持 JSON 输出
  - 是否支持 dry-run
  - 是否支持指定目标环境
  - 是否支持 list / install / remove / update
  - 输出格式是否稳定
  - 写操作会改哪些文件

  结果补进：

  docs/02-cli-capabilities.md

  ———

  ### 2. 明确 MVP 第一轮只做什么

  建议第一轮只做：

  1. 本地 Web UI 启动
  2. 后端调用 npx skills
  3. 展示 skills / environments
  4. 执行 install skill
  5. 执行 add-mcp
  6. 展示日志

  不要第一轮做：

  备份恢复
  复杂 diff
  桌面打包
  多客户端同步
  自定义 registry

  ———

  ### 3. 确认开发环境

  需要确认：

  node -v
  pnpm -v
  git --version

  建议：

  Node.js >= 20
  pnpm >= 9
  TypeScript

  ———

  ## 二、项目初始化工作

  建议初始化成轻量 monorepo：

  skilldock/
    apps/
      web/
      server/
    packages/
      shared/
    docs/
    package.json
    pnpm-workspace.yaml
    tsconfig.base.json
    .gitignore
    README.md

  ———

  ## 三、初始化顺序

  ### 1. Git 初始化

  cd ~/skilldock
  git init

  ———

  ### 2. 创建 pnpm workspace

  pnpm-workspace.yaml

  包含：

  packages:
    - "apps/*"
    - "packages/*"

  ———

  ### 3. 初始化根 package.json

  根项目只放公共脚本：

  {
    "name": "skilldock",
    "private": true,
    "type": "module",
    "scripts": {
      "dev": "pnpm -r --parallel dev",
      "build": "pnpm -r build",
      "lint": "pnpm -r lint"
    }
  }

  ———

  ### 4. 初始化 shared 包

  放共享类型：

  packages/shared/
    src/
      types.ts

  例如：

  export type CommandResult = {
    exitCode: number
    stdout: string
    stderr: string
    durationMs: number
  }

  ———

  ### 5. 初始化 server

  技术栈：

  Fastify + TypeScript + execa + pino

  先实现：

  GET /api/status
  POST /api/commands/skills-help
  POST /api/commands/add-mcp-help

  用于验证 CLI 调用链路。

  ———

  ### 6. 初始化 web

  技术栈：

  Vite + React + TypeScript + Tailwind

  先做：

  首页
  CLI 状态卡片
  命令输出面板

  ———

  Phase 1:

  ———

  ## 五、最小启动目标

  第一轮开发完成后，应该能做到：

  pnpm dev

  然后浏览器打开 Web UI，可以看到：

  SkillDock running
  npx skills 可用 / 不可用
  npx add-mcp 可用 / 不可用
  命令输出日志

  ———

  一句话：
  先验证 CLI 调用链路，再做 UI，不要一开始实现复杂业务。
